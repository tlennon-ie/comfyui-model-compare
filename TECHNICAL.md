# Technical Documentation - ComfyUI Model Compare

## Architecture Overview

ComfyUI Model Compare is a four-node pipeline system that enables comparison of multiple model configurations through a combination-driven sampling workflow.

```
┌─────────────────────────────────────────────────────────────────┐
│                    Model Compare Pipeline                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ Node 1: Model Compare Loaders (CONFIG)                  │   │
│  │ Purpose: Define how many models will be compared        │   │
│  │ Input: num_checkpoints, num_vaes, num_text_enc, num_loras │
│  │ Output: config dict template                            │   │
│  └──────────────┬───────────────────────────────────────────┘   │
│                 │ config                                         │
│                 ▼                                                 │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ Node 2: Model Compare Loaders Advanced (SELECT)         │   │
│  │ Purpose: Select specific models and LoRA strengths      │   │
│  │ Input: config + dynamic selection dropdowns             │   │
│  │ Output: config with combinations computed               │   │
│  │ Key: Generates Cartesian product of all selections      │   │
│  └──────────────┬───────────────────────────────────────────┘   │
│                 │ config (with combinations)                     │
│                 ▼                                                 │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ Node 3: Sampler Compare (SAMPLING)                      │   │
│  │ Purpose: Generate samples for all combinations          │   │
│  │ Input: config + latent + conditioning + sampling params │   │
│  │ Output: concatenated images + labels                    │   │
│  │ Process: Loop through combinations, load models, sample │   │
│  └──────────────┬───────────────────────────────────────────┘   │
│                 │ images, labels                                 │
│                 ▼                                                 │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ Node 4: Grid Compare (VISUALIZATION)                    │   │
│  │ Purpose: Arrange samples in customizable grid           │   │
│  │ Input: images + labels + styling parameters            │   │
│  │ Output: grid image + save path                          │   │
│  │ Process: PIL image composition with borders/labels      │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## Core Data Structures

### Configuration Dictionary

```python
config = {
    # Count specifications (user sets these in Node 1)
    "num_checkpoints": int,        # 0-10
    "num_vaes": int,               # 0-5
    "num_text_encoders": int,      # 0-5
    "num_loras": int,              # 0-10
    
    # Selected models (populated by Node 2)
    "checkpoints": [str, ...],     # e.g., ["model_a.safetensors", ...]
    "vaes": [str, ...],            # e.g., ["vae.safetensors", ...]
    "text_encoders": [str, ...],   # e.g., []
    "loras": [
        {
            "name": str,           # e.g., "detail.safetensors"
            "strengths": [float]   # e.g., [0.5, 1.0, 1.5]
        },
        ...
    ],
    
    # Computed combinations (generated by Node 2)
    "combinations": [
        {
            "checkpoint": str or None,
            "vae": str or None,
            "text_encoder": str or None,
            "lora_strengths": tuple or None,  # e.g., (0.5, 1.0)
            "lora_names": [str],               # Model names for reference
            "label": str                       # Human-readable label
        },
        ...
    ]
}
```

### Combination Generation Algorithm

```python
def _compute_combinations(config):
    """
    Generate Cartesian product of all model selections.
    
    Mathematical formula:
    Total combinations = |checkpoints| × |vaes| × |text_encoders| × 
                        (|lora_strength_1| × |lora_strength_2| × ...)
    
    Example:
    - 2 checkpoints
    - 1 VAE
    - 0 text encoders
    - 2 LoRAs with [0.5, 1.0] and [0.0, 1.0] strengths
    
    Result: 2 × 1 × 1 × (2 × 2) = 8 combinations
    """
    
    # Get base model lists (default to [None] if empty)
    checkpoints = config.get("checkpoints", []) or [None]
    vaes = config.get("vaes", []) or [None]
    text_encoders = config.get("text_encoders", []) or [None]
    
    # Get LoRA strength combinations
    loras = config.get("loras", [])
    if loras:
        lora_strength_lists = [l["strengths"] for l in loras]
        lora_strength_combos = itertools.product(*lora_strength_lists)
    else:
        lora_strength_combos = [None]
    
    # Generate all combinations using Cartesian product
    combinations = []
    for ckpt, vae, text_enc, lora_strengths in itertools.product(
        checkpoints, vaes, text_encoders, lora_strength_combos
    ):
        combination = {
            "checkpoint": ckpt,
            "vae": vae,
            "text_encoder": text_enc,
            "lora_strengths": lora_strengths,
            "lora_names": [l["name"] for l in loras] if loras else [],
            "label": _make_label(ckpt, vae, text_enc, loras, lora_strengths)
        }
        combinations.append(combination)
    
    return combinations
```

### 3. SamplerCompare

**Purpose**: Execute sampling across all combinations

**Key Algorithm**:
```python
for combination in config["combinations"]:
    # Step 1: Load models for this combination
    if combination["checkpoint"]:
        model, clip, vae = load_checkpoint(combination["checkpoint"])
    else:
        # Use provided models
        pass
    
    if combination["vae"]:
        vae = load_vae(combination["vae"])
    
    # Step 2: Apply LoRAs
    if combination["lora_names"] and combination["lora_strengths"]:
        for lora_name, strength in zip(
            combination["lora_names"],
            combination["lora_strengths"]
        ):
            model, clip = apply_lora(model, clip, lora_name, strength)
    
    # Step 3: Sample
    latent_sampled = ksampler(
        model, latent, seed + idx, cfg, steps, ...
    )
    
    # Step 4: Decode
    image = vae.decode(latent_sampled)
    all_images.append(image)
```

### 4. GridCompare

**Purpose**: Create visual comparison grid

**Grid Calculation**:
```python
# Determine dimensions
num_checkpoints = len(config["checkpoints"])
cols = max(1, num_checkpoints)
rows = ceil(len(images) / cols)

# Grid dimensions
grid_width = cols * (img_width + gap) + gap + label_width
grid_height = rows * (img_height + gap) + gap + 2*label_height + title_height
```

**Grid Composition Algorithm**:
```python
# Create blank grid image
grid = Image.new('RGB', (grid_width, grid_height), color=bg_color)
draw = ImageDraw.Draw(grid)

# Draw title
draw.text(center, title, fill=text_color)

# For each image
for idx, image in enumerate(images):
    row = idx // cols
    col = idx % cols
    
    x = label_width + gap + col * (img_width + gap)
    y = title_height + label_height + gap + row * (img_height + gap)
    
    # Draw border
    if border_width > 0:
        draw.rectangle([x, y, x+img_width, y+img_height], 
                      outline=border_color, width=border_width)
    
    # Paste image
    grid.paste(image, (x, y))
    
    # Draw label above image
    draw.text((x, y-label_height//2), labels[idx], 
             fill=text_color, font=font)
```

## Key Functions

### Combination Label Generation

```python
def _make_label(checkpoint, vae, text_encoder, loras, lora_strengths):
    """
    Generate human-readable label for a combination.
    
    Format:
    "ckpt:model_v1 | vae:vae_a | lora_detail:0.50 | lora_boost:1.00"
    
    Includes:
    - Checkpoint basename (without extension)
    - VAE basename
    - Text encoder basename (if any)
    - LoRA names with strength values (2 decimal places)
    """
    parts = []
    
    if checkpoint:
        parts.append(f"ckpt:{os.path.splitext(checkpoint)[0]}")
    
    if vae:
        parts.append(f"vae:{os.path.splitext(vae)[0]}")
    
    if text_encoder:
        parts.append(f"enc:{os.path.splitext(text_encoder)[0]}")
    
    if loras and lora_strengths:
        for lora, strength in zip(loras, lora_strengths):
            lora_name = os.path.splitext(lora["name"])[0]
            parts.append(f"{lora_name}:{strength:.2f}")
    
    return " | ".join(parts) if parts else "default"
```

### Model Loading

```python
def _load_checkpoint(checkpoint_name: str) -> Tuple[Model, CLIP, VAE]:
    """
    Load a checkpoint using ComfyUI's built-in loader.
    
    Returns:
    - model: The diffusion model
    - clip: The text encoder
    - vae: The VAE encoder/decoder
    """
    ckpt_path = folder_paths.get_full_path("checkpoints", checkpoint_name)
    return comfy.sd.load_checkpoint_guess_config(ckpt_path, ...)

def _load_loras(
    model: Model,
    clip: CLIP,
    lora_names: List[str],
    strengths: Tuple[float, ...]
) -> Tuple[Model, CLIP]:
    """
    Load and apply LoRAs to model and clip with specified strengths.
    
    LoRA strength controls the influence of the LoRA:
    - 0.0: No effect
    - 1.0: Full LoRA influence
    - 1.5+: Amplified LoRA effect
    """
    for lora_name, strength in zip(lora_names, strengths):
        lora_path = folder_paths.get_full_path("loras", lora_name)
        lora = comfy.sd.load_lora(lora_path)
        model, clip = comfy.sd.load_lora_for_models(
            model, clip, lora, strength, strength
        )
    return model, clip
```

## Performance Considerations

### Memory Usage

```
Per sample memory = Model + CLIP + VAE + Latent workspace
Typical: ~12-24 GB VRAM for modern models

Total time = sum(load_time + sample_time + decode_time) for each combination

Example: 4 combinations, 25 steps each
- Load model: 1-2 seconds per load
- Sample: 10-15 seconds per sample
- Decode: 1 second per VAE decode
- Total: ~60-100 seconds for 4 samples
```

### Optimization Strategies

1. **Model Caching**:
   - Keep model in VRAM between samplings if same checkpoint
   - Current implementation: Loads each time

2. **LoRA Caching**:
   - Pre-load all LoRAs, apply/unapply as needed
   - Current implementation: Loads each time

3. **Batch Sampling**:
   - Group samples with same model/VAE
   - Current implementation: Sequential processing

## Error Handling

### Common Errors and Recovery

```python
# Missing model file
if not os.path.exists(model_path):
    print(f"[SamplerCompare] Error: {model_name} not found")
    continue  # Skip this combination

# Invalid LoRA strength format
try:
    strengths = [float(s.strip()) for s in strengths_str.split(",")]
except ValueError:
    print(f"[ModelCompareLoadersAdvanced] Invalid strengths: {strengths_str}")
    strengths = [1.0]  # Fallback to default

# Empty image list
if not all_images:
    empty_tensor = torch.zeros((1, 64, 64, 3))
    return (empty_tensor, "No successful samples")
```
