## ComfyUI Widget System

### Widget Types
```python
# Basic types
"param_name": ("STRING", {"default": "value"})
"param_name": ("INT", {"default": 0, "min": 0, "max": 100, "control_after_generate": True})
"param_name": ("FLOAT", {"default": 7.5, "min": 0.0, "max": 10.0, "step": 0.1})
"param_name": ("COMBO", {
    "default": "option1",
    "options": ["option1", "option2", "option3"]
})

# Special behaviors
"control_after_generate": True  # Updates after execution
type: "click"  # Trigger action
type: "toggle"  # Checkbox
```

### Configuration Layout Patterns
```python
# Group related controls
"sampler_name": ("COMBO", {...}),  # Use standard ComfyUI sampler list
"scheduler": ("COMBO", {...}),     # Use standard ComfyUI scheduler list
"steps": ("INT", {"default": 20, "min": 1, "max": 200}),
"cfg": ("FLOAT", {"default": 7.5, "min": 0.0, "max": 15.0, "step": 0.1}),
```

### Frontend Communication
- Widgets are defined in `INPUTS` dict
- Values come from ComfyUI's widget system
- Special fields: `control_after_generate`, `tooltip`, `display`
- Node execution returns `(output_data), ui_updates_dict`

**Pattern**: Multi-model picker with clear naming

**Pattern**: Grouped controls organized by concern

### 4. Grid Rendering
**Pattern**: Clear visual comparison layout

- Show labels with key parameters
- Handle varying dimensions gracefully
- Include legend or key for parameters

### 5. Progress & Feedback
**Pattern**: Clear status updates

## Widget Design Principles for This Node

### Principle 1: Progressive Disclosure
**Goal**: Beginner users see simple controls, advanced users access complex options

```python
INPUTS = {
    # Basic tier - always visible
    "models": ("MODEL_LOADER", {}),
    "positive_prompt": ("STRING", {
        "multiline": True,
        "default": "a beautiful woman"
    }),
    "negative_prompt": ("STRING", {...}),
    "seed": ("INT", {"control_after_generate": True}),
    
    # Intermediate - in an expandable section
    "sampler_options": ("STRING", {"multiline": True}),  # Comma-separated values
    
    # Advanced - hidden by default
    "enable_loras": ("COMBO", {"options": ["No", "Yes"]}, {}),
}
```

### Principle 2: Smart Defaults
**Goal**: Users get good results without deep configuration

```python
# Auto-select common sampler/scheduler combos
"sampler_name": ("COMBO", {
    "default": "euler",  # Reliable general-purpose
    "options": ["euler", "DPM++ 2M", "DPM++ SDE"]
}),

# Reasonable CFG range
"cfg": ("FLOAT", {
    "default": 7.5,  # Works for most models
    "min": 0.0,
    "max": 15.0,
    "step": 0.5
}),
```

### Principle 3: Guided Variations
**Goal**: Make it obvious how to create parameter variations

```python
"steps": ("STRING", {
    "default": "20",
    "tooltip": "Single value or comma-separated: 20,30,40"
}),

"seed_control": ("COMBO", {
    "options": ["fixed", "increment", "randomize"],
    "default": "fixed",
    "tooltip": "How to change seed after current run completes"
}),
```

### Principle 4: Validation Feedback
**Goal**: Tell users what went wrong in natural language

```python
# Good error message
"ERROR: 8+ combinations estimated at 400s (threshold 120s). Consider fewer variations."

# Show warning for high load
"⚠️ High combination count: 16 combinations. Estimated time: 250 seconds."
```

### Principle 5: Visual Organization
**Goal**: Related controls grouped, clear visual hierarchy

```python
# Example node layout for SamplerCompareAdvanced
[Model Selection]          # Compact model picker
[Prompt Inputs]            # Large text areas
[Sampling Controls]        # Seed, Steps, CFG, Sampler grouped
[Variation Options]        # Enable multi-value variations
[Model-Specific Controls]  # Advanced, collapsed by default
[LoRA Configuration]       # If LoRA Compare present
[Output Options]           # Grid format, export settings
```

## Common UX Issues & Solutions

### Issue 1: Seed Widget Not Updating
**Problem**: Seed value doesn't change after generation  
**Solution**: Use INT with `control_after_generate: True`
```python
"seed": ("INT", {
    "default": 0,
    "min": 0,
    "max": 2**32 - 1,
    "control_after_generate": True
})
```

### Issue 2: Too Many Parameters, UI is Overwhelming
**Problem**: Node has 20+ widgets, hard to understand  
**Solution**: Group related controls, use collapsible sections
```python
# Group by concern
"std_seed": ("INT", {...}),  # Standard seed
"std_steps": ("INT", {...}),

"adv_denoise": ("FLOAT", {...}),    # Advanced
"adv_cfg_norm": ("COMBO", {...}),
```

### Issue 3: Complex Multi-Value Input
**Problem**: Users don't understand comma-separated format  
**Solution**: Clear tooltip and examples
```python
"sampling_variations": ("STRING", {
    "default": "",
    "multiline": True,
    "tooltip": "Examples:\nsampler_name: euler,DPM++\nsteps: 20,30,40\nSchedule values with # separator"
})
```

### Issue 4: Grid Output Not Readable
**Problem**: Images too small, labels cut off  
**Solution**: Scalable grid rendering, responsive sizing
- Include zoom controls
- Auto-scale based on viewport
- Readable label placement

### Issue 5: No Feedback During Long Generation
**Problem**: User thinks it crashed  
**Solution**: Real-time progress updates
- Current combination (X of Y)
- Estimated time remaining
- Memory usage
- Cache hit rate
