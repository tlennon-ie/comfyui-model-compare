# comfyui-model-compare Agent Suite

A comprehensive set of specialized agents designed to guide development of the comfyui-model-compare custom node across all domains: product, engineering, and user experience.

## Agents Overview

### 🎯 `.product-manager.agent.md`
**Feature strategist** for user workflows and release quality  
**When to use**: Planning features, validating requirements, assessing product readiness

**Handles**:
- Feature planning and prioritization based on user needs
- Use case validation and workflow design
- Model support strategy and compatibility
- Release quality assurance and documentation
- Integration with ComfyUI ecosystem

**Example prompts**:
- "What features do users need most for model comparison?"
- "Is [feature] ready for production release?"
- "Plan support for [new model type within ComfyUI]"
- "What's blocking release quality?"

## Agent Collaboration Patterns

### New Feature Development
1. **Product Manager**: Define use case and acceptance criteria
2. **Engineer**: Design technical approach and architecture
3. **UX Engineer**: Design widget layout and user workflows
4. **Engineer**: Implement the feature
5. **UX Engineer**: Validate UI/UX experience
6. **Product Manager**: Assess release readiness

### Bug Fix Process
1. **Engineer**: Diagnose root cause
2. **UX Engineer**: Identify if UX contributed to bug (e.g., confusing parameter)
3. **Engineer**: Implement fix
4. **Product Manager**: Validate fix doesn't break workflows

### Performance Optimization
1. **Engineer**: Profile and identify bottlenecks
2. **Product Manager**: Validate impact on user workflows
3. **Engineer**: Implement optimization
4. **UX Engineer**: Surface performance insights to users

### Model Type Support
1. **Product Manager**: Assess user demand and use cases
2. **Engineer**: Research ComfyUI integration requirements
3. **Engineer**: Implement model-specific logic
4. **UX Engineer**: Design widget controls for model-specific parameters
5. **Product Manager**: Validate completeness and prepare documentation

## Release Readiness Checklist

Before each release, all three agents validate:

**Product Manager**:
- [ ] Feature complete per spec
- [ ] User documentation current
- [ ] Known limitations documented
- [ ] Backward compatibility validated
- [ ] All supported models tested

**Engineer**:
- [ ] No deprecation warnings from ComfyUI
- [ ] Performance acceptable (8 combos < 60s)
- [ ] No unbounded resource growth
- [ ] Error handling comprehensive
- [ ] Tests passing

**UX Engineer**:
- [ ] Widgets intuitive and well-organized
- [ ] Help text clear and accessible
- [ ] Progress feedback visible
- [ ] Grid rendering correct for all dimensions
- [ ] Mobile/narrow viewport support

## How to Invoke These Agents

### In VS Code
When chatting with GitHub Copilot, reference the agent directly:
- "Ask @product-manager: What features should we prioritize?"
- "@engineer: Can you debug this error?"
- "@ux-engineer: Design the widget layout for seed control"

### In GitHub Copilot Chat (Web)
Use the `@agent` mention syntax:
```
@product-manager Should we add support for [model type]?
```

### Command Line (with Extensions)
```bash
copilot --agent product-manager --prompt "What's blocking release?"
copilot --agent engineer --prompt "Fix: UnboundLocalError in use_seed"
copilot --agent ux-engineer --prompt "Design progress feedback UI"
```
