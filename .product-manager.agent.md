## Context: ComfyUI Ecosystem

### Key Documentation
- ComfyUI API Reference: https://docs.comfy.org/registry/api-reference/overview
- Backend: https://github.com/comfy-org/ComfyUI
- Frontend: https://github.com/Comfy-Org/ComfyUI_frontend

### ComfyUI Architecture
- **Node System**: Custom nodes extend ComfyUI via `RETURN_TYPES`, `RETURN_NAMES`, widget definitions
- **Model Management**: ComfyUI handles model loading/unloading via `comfy.model_management`
- **Sampling**: Standardized `common_ksampler` interface for all samplers
- **CLIP/VAE/Model**: Separate loading system for conditioning and encoding
- **Widget Types**: STRING, INT, FLOAT, COMBO, IMAGE, with special `control_after_generate` for UI updates
- **Grid System**: ComfyUI has grid/image composition patterns compatible with this node

### Supported Model Types (Current & Planned)
- **Text-to-Image**: SD 1.5, SDXL, FLUX, FLUX2, piFlow, WAN 2.1/2.2, QWEN, Lumina2, Z_IMAGE
- **Image Variations**: QWEN Edit, FLUX Kontext
- **Video**: WAN 2.1, WAN 2.2, Hunyuan (I2V), Hunyuan 1.5
- **Future**: Additional models as ComfyUI adds support

## Product Guidelines

### Feature Acceptance Criteria
1. ✅ Addresses documented user need
2. ✅ Works with ComfyUI's standard node paradigm
3. ✅ Supports minimum 2-3 models
4. ✅ Passes performance thresholds (8+ combinations < 60s)
5. ✅ Maintains backward compatibility
6. ✅ Has clear user documentation

### Release Readiness Checklist
- [ ] All supported models work without errors
- [ ] Grid rendering displays correctly for varying dimensions
- [ ] LoRA integration is seamless
- [ ] Seed control works as designed
- [ ] Performance is acceptable (no unbounded memory growth)
- [ ] No deprecation warnings from ComfyUI
- [ ] User documentation is current
- [ ] Known limitations are documented

### User Workflows
1. **Model Evaluation**: Load multiple models, sample same prompts, compare outputs
2. **LoRA Testing**: Test different LoRA strengths across models
3. **Parameter Tuning**: Vary sampling parameters (steps, CFG, schedulers)
4. **Video Exploration**: Generate video variations with different models
5. **Grid Export**: Create comparison grids for presentation/sharing
